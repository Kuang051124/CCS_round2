# 项目指引：基于 MSPM0G3507 的自主循迹与激光瞄准小车

## 概述

这是一个 **TI MSPM0G3507 单片机**（ARM Cortex-M0+ 内核）竞赛小车项目，运行在 **LP-MSPM0G3507 LaunchPad** 开发板上。小车具备**自主循迹**、**视觉目标检测**、**激光瞄准**和**陀螺仪导航**能力。

## 硬件架构

### 运动系统
| 组件 | 驱动芯片 | 说明 |
|------|----------|------|
| 4 路直流有刷电机 (FL/FR/BL/BR) | TB6612 | 四轮驱动底盘，带编码器测速，支持 PID 速度/位置闭环 |
| 水平步进电机 | TB6600 | 控制激光云台水平旋转（TIMG6, PA29） |
| 垂直步进电机 | TB6600 | 控制激光云台俯仰旋转（TIMA1, PA28） |

### 传感器系统
| 传感器 | 通信接口 | 用途 |
|--------|----------|------|
| WIT 陀螺仪模块 | UART0 (PA11 RX, 115200bps, DMA) | 获取 yaw/pitch/roll 姿态角，用于角度闭环控制 |
| BNO08x IMU | UART RVC 模式 | 备用姿态传感器 |
| MPU6050 (含 DMP) | I2C | 备用 6 轴姿态传感器 |
| LSM6DSV16X | I2C | 备用 6 轴 IMU |
| 5 路灰度循迹模块 | GPIO (PB12/PB10/PB8/PB6/PB7) | 黑线循迹，检测直线和直角弯 |
| 8 路灰度循迹模块 | GPIO | 备选高密度循迹方案 |
| VL53L0X ToF 激光测距 | I2C | 距离测量 |
| 超声波模块 | GPIO / 输入捕获 | 避障/测距（两种驱动方案） |

### 人机交互
| 组件 | 接口 | 说明 |
|------|------|------|
| OLED 显示屏 | I2C1 (PB3 SDA / PB2 SCL) | 128x64 点阵显示，支持 4 种驱动（软/硬件 I2C/SPI） |
| 3×3 矩阵键盘 | GPIO (PA17/PA9/PB13 ROW, PA16/PA14/PA13 COL) | 菜单导航与参数调节 |
| 蜂鸣器 | GPIO (PA30) | 操作反馈提示音 |
| 紫色激光笔 | GPIO (PA8) | 瞄准指示 |

### 视觉与通信
| 组件 | 接口 | 说明 |
|------|------|------|
| K230 AI 摄像头 | UART2 (PB18 RX / PB17 TX, 115200bps) | 视觉识别目标位置和激光点位置 |
| 自定义协议 (yb_protocol) | 串口 | 帧头 `$`(0x24)、帧尾 `#`(0x23)，解析 K230 下发的坐标数据 |

### 主程序逻辑

程序使用 **页面状态机** 进行调度 (`page_state` 0~6)，通过 `main.c` 中的无限循环分发：

| 页面 | 功能 |
|------|------|
| **0 - Home** | 主菜单，选择进入校准/调试/测试页面 |
| **1 - Calib** | 动态参数校准：直走占空比、转弯占空比、转弯延时、拐弯成功次数 |
| **2 - Debug** | 调试菜单：步进电机测试、陀螺仪校准、激光开关、循迹值查看、K230 坐标接收 |
| **3 - Test** | 核心测试任务（见下方） |
| **4 - Debug Step** | 步进电机微调：水平/垂直手动控制 |
| **5 - Debug Gyro** | 陀螺仪校准与角度 PID 对准 |

### Test 页面（page_state=3）核心任务

这是竞赛任务的核心执行页面，通过键盘按键触发：

| 按键 | 任务 | 说明 |
|------|------|------|
| **1** | 循迹绕圈 | 5 路循迹跟随黑线，检测直角转弯并计数，绕指定圈数后停止 |
| **2** | 激光瞄准（固定朝向） | K230 识别目标+激光点坐标，PID 控制步进电机将激光对准目标 |
| **3** | 激光瞄准（旋转 + 瞄准） | 先手动指定方向（顺/逆时针），陀螺仪控制旋转到位后，再 PID 瞄准 |
| **4** | 激光瞄准（逆时针寻找） | 初始逆时扫描寻找目标，检测到后用 PID 锁定目标，中靶多次确认 |
| **5** | 激光瞄准（顺时针寻找） | 与按键 4 同理，顺时针方向扫描寻找 |
| **6** | 发挥模式：循迹 + 瞄准 | 边走线边用陀螺仪保持朝向，实时接收 K230 目标坐标预测跟踪 |

### 控制算法

- **循迹**：5 路灰度传感器加权求和，根据偏差值调整左右轮差速
- **直角转弯检测**：当所有传感器检测到黑线时触发转弯；转弯后检测中间传感器恢复到黑线来确认转弯完成
- **激光瞄准**：K230 通过串口下发 `(target_x, target_y, laser_x, laser_y)` 坐标，MSPM0 用**位置式 PID** 分别控制水平和垂直步进电机，缩小激光点与目标的距离
- **陀螺仪旋转**：WIT 模块提供 yaw 角，计算最短弧方向，用步进电机旋转到目标朝向

## 开发环境

- **IDE**：TI Code Composer Studio (CCS) + SysConfig
- **SDK**：MSPM0 SDK v2.05.00.05
- **编译器**：TI ARM Clang Compiler
- **目标芯片**：MSPM0G3507 (LQFP-64 封装)
- **调试器**：XDS-110 (通过 LP-MSPM0G3507 板载)

## 编译与烧录

1. 在 CCS 中导入项目
2. 打开 `Target.syscfg` 确认引脚配置
3. 编译项目（Project → Build All）
4. 通过 XDS-110 烧录并调试（Run → Debug）

## 注意事项

- 调试时需保持 `J101 15:16 ON`（SWCLK）和 `J101 13:14 ON`（SWDIO）跳线连接
- 若应用中使用 PA19/PA20，调试完成后需断开对应跳线
- 步进电机和直流电机的 PID 参数可在 Calib 页面实时调节
- K230 摄像头必须与 MSPM0 共地，通信协议以 `$` 开头、`#` 结尾
- 代码中包含 5 路和 8 路两套循迹方案，以及多套电机占空比方案，通过注释切换
